플랫폼은 서비스고, 프론트엔드는 도구예요. 플랫폼은 회사가 망하거나 정책이 바뀌면 데이터도 캐릭터도 같이 흔들려요. 프론트엔드는 오픈소스라서 로컬이든 셀프호스팅이든 통제권이 그대로 남아있어요. 대신 그만큼 세팅도 직접 챙겨야 하는 거예요.

03 · 가이드 목차

천천히 하나씩 볼까요

순서대로 안 봐도 괜찮아요. 지금 막힌 부분을 눌러서 열어보세요 👀

RisuAI는 기본적으로 웹사이트로 사용해요. 크롬, 사파리, 삼성 인터넷 같은 브라우저에서 바로 접속할 수 있어요. 이 외에도 포켓리수, 노드리수가 있어요. 처음에는 웹사이트로 하는 걸 권장해요. API 키는 제미나이, 클로드 같은 곳에서 발급받아 설정 > API 항목에 등록하면 돼요. 키를 여러 개 등록해두고 상황에 따라 전환하는 것도 가능해요. API 키 연결이랑 설치 방법은 초보자 가이드 글에 자세히 정리되어 있으니, 꼭 먼저 확인해주세요.
캐릭터 카드는 이름, 설명, 성격, 첫 메시지, 예시 대화 같은 필드로 구성돼요. 로어북은 세계관처럼 부피가 큰 설정을 카드 밖으로 빼두는 저장소예요. 특정 키워드가 등장할 때만 불러오고, depth와 우선순위로 언제 어떤 정보가 노출될지 조절해요. 카드는 캐릭터의 핵심만 담아 가볍게 유지하고, 설정은 로어북으로 확장하는 구조예요. 여기에 작가의 노트를 더하면, 매 턴마다 AI에게 짧은 지시(말투 유지, 특정 행동 유도 등)를 얹어줄 수 있어요. 로어북이 캐릭터에게 "정보"를 주는 거라면, 작가의 노트는 AI에게 "지시"를 주는 느낌이에요.
하이파 메모리는 리수에서 쓸 수 있는 부가 기능이에요. 별도로 가입만 하면 평생 무료로 쓸 수 있어요. 컨텍스트를 넘어가는 내용을 요약해서 저장해주기 때문에 장기 기억이 확 늘어나요.
CSS/HTML로는 테마 색상, 채팅 버블 모양, 상태창 UI 같은 화면 요소를 원하는 대로 꾸밀 수 있어요. 다른 사람이 만든 프리셋을 가져와서 내 캐릭터에 맞게 수정하는 것도 가능하고, 코드가 낯설어도 기존 프리셋을 그대로 가져다 쓰는 정도는 어렵지 않아요.
Lua 스크립트는 카드나 로어북이 담당하는 '설정'이 아니라, 입출력을 실시간으로 가공하는 '코드' 영역이에요. editDisplay 함수는 화면에 보이는 텍스트를 실제 대화 기록과 다르게 표시해주고, onOutput 훅은 AI 응답이 나올 때마다 실행돼서 원하는 치환·포맷 로직을 넣을 수 있어요. 상태창이나 자동 마크다운 제거 같은 기능도 이 두 가지 조합으로 만들어져요. 다만 초보자에게는 난이도가 있는 영역이에요. 직접 만들기는 쉽지 않고, 안 쓴다고 캐릭터 대화 자체에 문제가 생기는 것도 아니니 굳이 무리해서 배우지 않으셔도 괜찮아요.
V3 플러그인 포맷은 IIFE(즉시실행함수)로 코드 전체를 감싸서 전역 스코프 오염을 막는 구조예요. 플러그인 안에서 UI를 추가하고 싶으면 플로팅 패널을 DOM에 직접 삽입하고, 체크박스나 버튼 같은 인터랙션 요소를 붙여서 클립보드 복사 같은 기능을 구현할 수 있어요. 이 역시 초보자에게는 난이도가 있는 기능이에요. 직접 만들기는 어렵고, 굳이 쓰지 않아도 되는 기능이니 필요할 때만 참고하셔도 돼요.
