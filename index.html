01 — CORE DIFFERENCE

플랫폼 vs 프론트엔드

둘 다 "AI랑 캐릭터 채팅한다"는 결과는 같다. 그런데 스위치판을 누가 쥐고 있느냐가 완전히 다르다.

PLATFORM 제타 · 크랙
  • 모델 선택 — 회사가 지정
  • 프롬프트 구조 — 비공개
  • 컨텍스트 크기 — 고정
  • 과금 방식 — 정액 / 재화
  • 데이터 위치 — 회사 서버
  • 진입장벽 — 앱 켜면 끝

잠긴 스위치가 많을수록, 대신 아무것도 몰라도 된다.

FRONTEND RisuAI
  • 모델 선택 — API 직접 연결
  • 프롬프트 구조 — 전체 수정 가능
  • 컨텍스트 크기 — 직접 조절
  • 과금 방식 — 사용한 토큰만큼
  • 데이터 위치 — 로컬 / 자체호스팅
  • 진입장벽 — API 키부터 구해야 함

전부 열려 있는 대신, 전부 니가 설정해야 한다.

기억해둘 것 플랫폼은 서비스고, 프론트엔드는 도구다. 플랫폼은 회사가 망하거나 정책을 바꾸면 니 데이터도 니 캐릭터도 같이 흔들린다. 프론트엔드는 오픈소스라 로컬에 있든 셀프호스팅하든 통제권이 너한테 남아있다. 대신 그 통제권만큼 세팅의 책임도 너한테 있다.

02 — WHAT IS RISUAI

RisuAI, 뜯어보면

이름은 블루 아카이브 Arisu에서 A를 뺀 것. 한국인 개발자가 커뮤니티 개발로그로 시작한 오픈소스 프로젝트다.

SETUP

세 가지 설치 방식

웹사이트 접속, 로컬 설치, Docker 자체호스팅. Node.js 로컬 서버 실행도 지원한다.

MODEL

모델은 니가 물어온다

OpenAI 호환 포맷이면 전부 연결 가능. OpenRouter·mancer 통해 DeepSeek, Qwen, Kimi 같은 마이너 모델까지, 물론 Claude·GPT·Gemini도.

MEMORY

하이파 메모리

컨텍스트를 넘어가는 내용을 요약해서 저장. AI를 하나 더 돌리는 셈이라 비용은 늘지만 장기 기억이 확 늘어난다.

LOREBOOK

로어북 & 랜덤 이벤트

세계관 설정, 조건부 발동, 이벤트 삽입까지 — 캐릭터 밖의 "무대"를 짜는 영역.

SCRIPT

Lua · JS 플러그인

정규식, Lua 스크립트, JS 플러그인으로 입출력을 전처리/후처리. 상태창, 자동 마크다운 제거, 톤 변환까지 전부 이 레이어에서 만든다.

SHARE

RisuRealm

캐릭터 카드(.charx) 공유·다운로드 서비스. 웹 버전에 바로 불러오거나 JSON으로 받을 수 있다.
